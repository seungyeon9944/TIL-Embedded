## 충돌 회피
### 1. 맵, 경로 기반 충돌 회피
- 맵, 경로 데이터를 활용해 충돌하지 않고 주행하는 방법
- 이미 알고 있는 공간을 지날 때 회피하는 알고리즘 (스마트홈 로봇, 자율주행차)
    - **Dijkstra**(맵) - 모든 노드를 탐색해 목적지까지 잇는 모든 거리의 총합이 최소가 되는 경로를 찾는다
    - Lattice planner(경로)

### 1-1. Dijkstra를 이용한 Path Planning
![Cost Map](https://i.imgur.com/WD7mZHB.png)
- **Global Cost Map**을 이용하여 전역 경로를 획득하는 방법
1. 로봇의 공간 좌표(x,y)를 Global Cost Map에 대한 grid 좌표 변환
2. 목적지 grid를 입력
3. Dijkstra 알고리즘을 이용해 최단 경로를 얻음
4. 탐색한 경로는 grid 좌표로 얻기 때문에 우리가 사용하고 있는 공간 좌표(x,y)로 변환

- **Local Cost Map**을 이용하여 지역 경로를 획득하는 방법
1. 전역경로를 Local Cost Map에 대한 grid 좌표로 변환
2. 변환한 경로점들의 cost를 확인해서 충돌 여부 판단
3. 충돌시 Dijkstra를 이용해 경로 탐색
4. 탐색한 경로는 grid 좌표로 얻기 때문에 우리가 사용하고 있는 공간 좌표(x,y)로 변환

### 1-2. Lattice planner
**격자(Lattice)** 를 이용해 충돌 회피하는 방법
- 기존 경로(Reference Path)를 이용해 다양한 조건으로 후보 경로 만듦
- 후보 경로 중 안전, 부드러움, 일관성 등을 고려해 주행할 경로를 선택

### 2. 센서 기반 충돌 회피
- 오로지 현재 측정되는 장애물인지 센서(라이다, 카메라, 초음파, 적외선)를 이용해서 충돌하지 않고 주행하는 방법
- 모르는 공간을 지날 때 회피하는 알고리즘 (미로, 탐사)
    - Wall following

### 2-1. Wall following
거리 센서(초음파, 라이다, 적외선)를 이용해 벽과 일정 거리를 유지하며 주행하는 로봇. 주변 환경에 따라 다양한 상황을 고려해야 함
